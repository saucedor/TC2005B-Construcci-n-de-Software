Preguntas respondidas

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
    Internet es la infraestructura física (red de redes). La WWW es un servicio que se transmite sobre Internet y permite acceder a páginas web mediante navegadores.
  2. ¿Cuáles son las partes de una URL?
    Esquema (https), dominio (www.ejemplo.com), puerto (opcional), ruta (/pagina), parámetros (?id=1) y ancla (#seccion).
  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    - GET: obtener recursos
    - POST: enviar datos para crear
    - PUT: reemplazar recurso
    - PATCH: modificar parcialmente
    - DELETE: eliminar recurso
    - HEAD: como GET, pero solo devuelve los headers
  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML con usuario y contraseña?
    POST, ya que no expone los datos en la URL.
  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página por URL?
    GET
  6. ¿Un servidor web devuelve una respuesta HTTP con código 200, qué significa eso?
    Que la solicitud fue exitosa y el contenido se entregó sin errores.
  7. ¿Es responsabilidad del desarrollador corregir si un usuario reporta error 404?
    Sí, significa que el recurso solicitado no existe o cambió su URL.
  8. ¿Y si el error es 500?
    También, ya que es un error del servidor o del código backend.
